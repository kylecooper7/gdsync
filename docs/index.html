gdsync

Let AI agents read and edit Google Docs through a simple CLI. No API complexity, no token juggling — just fetch, edit, commit.

Download Skill View on GitHub

How it works

gdsync syncs a Google Doc to a local file. The agent edits the file, runs a command, and changes appear in the live document.

gdsync fetch    # pull Google Doc into content.txt
# agent edits content.txt
gdsync commit   # push changes to the live doc

The sync engine handles all the complexity — Google Docs API, formatting, character indexes, named ranges, images, tables, lists, and comments.

Agent-friendly auth

Non-blocking two-step setup. No background processes or interactive prompts.

Your own GCP project

Each user gets their own Google Cloud project. gdsync never accesses your files.

Full document editing

Paragraphs, headings, lists, tables, images, inline formatting — all via clean markdown.

Comments

Read, reply to, and create comment threads. Agents can address feedback directly.

Suggestions mode

Visual tracked changes — old text strikethrough, new text in red. Let users review before accepting.

Doc discovery

List and search accessible docs, including shared files. No document ID needed.

Quick start

npm install -g @gdsync/gdsync

gdsync setup          # create GCP project (guided wizard)
gdsync setup check    # confirm setup
gdsync auth           # sign in for document access
gdsync auth check     # confirm sign-in
gdsync docs           # list available docs
gdsync init --doc ID  # start editing
